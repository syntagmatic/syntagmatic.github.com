Original
Similar to the d3 example. The changes are:

  • chord is its own function, taking data as its argument
  • clicking "Update" will draw a new chord diagram, with random data

Chart
A cleaner interface for updating and rendering the chart

  • chord returns an object
  • chord takes a container as its argument, so it can be for multiple charts
  • chord.update, chord.render and chord.clear work for each chart

Arc Tweens
Arcs are tweened for updates.

Chord and Tick Tweens
Chord and tick tweens for updates.

Fill Options
chord takes an object as its argument, with a mandatory container attribute and optional fill attribute

Remaining Issues

  • Chords "flip around" during update
  • Zero values break transition
